Quick Start

This is a quick start guide for getting up and running with Buck. To make it easy to get started, Buck provides a command, buck quickstart, which creates a simple project that is configured to be built with Buck out of the box.

If you are uncomfortable using a command to generate a project for you, then you may want to go through the manual quick start guide, which provides step-by-step instructions for creating your first project. The contents and purpose of each file needed to bootstrap the project are explained in the guide.

Step 1: Install Buck

As these are the "Quick Start" instructions, we assume that you already have Buck up and running. If you don't, please follow the instructions in Downloading and Installing Buck.

Step 2: Create an empty project

We should start our project in an empty directory, with an empty .buckconfig, so create a new directory, navigate to it, and run:
mkdir my-project && cd my-project && touch .buckconfig

Step 3: Fill in the empty project with a template

Now you can use buck quickstart to load a template into your project. There are two templates: an Android template and an iOS template. Load the Android template into the current directory by running
buck quickstart --type android --dest-dir .
or load the iOS template by running
buck quickstart --type ios --dest-dir .
You may be prompted for additional information. If the command succeeds, you will see some output explaining how to build and run the app in the template. If you wish to read the message again, just check the README.md file in your project directory.
